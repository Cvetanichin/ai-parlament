# 21 — Architecture Decision Records

Every change to the EAS or an approved detail spec is proposed here first as
`NNNN-title.md` (sequential numbering), following: Context → Decision → 
Consequences → Status (Proposed / Accepted / Superseded). Claude (Cowork) drafts,
the Product Owner approves, only then does the EAS or detail spec get amended.
No ADR, no architecture change — including changes proposed by Claude Code during
implementation if it discovers the spec doesn't fit reality.
